WORDPRESS — PUNT 1 + 2 AF (plugin 964d1ac)
--------------------------------------------
Live na deploy materialdistrict-plugin:

  GET  /md/v2/dashboard/profile
  POST /md/v2/dashboard/profile
  GET  /md/v2/dashboard/profile-options   ← nieuw

Profile GET/POST velden (snake_case):
  first_name, last_name, email, phone, profession, industry, address,
  postcode, city, country, invoice_to_company, company, vat_number, avatar_url

Usermeta-mapping (geen data gewist, legacy blijft leesbaar):
  phone              → telephone
  profession         → profession      (slug; legacy "Architect" → architect)
  industry           → sector          (slug; legacy "Interior" → interior)
  address            → address_street + address
  postcode           → postcode + zipcode
  city               → city
  country            → country         (ISO-code, bv. NL)
  invoice_to_company → billing_is_company ('true'/'false')
  company            → billing_company_name / company
  vat_number         → billing_vat_number

profile-options shape (zoals gevraagd):
  {
    "professions": [ { "value": "architect", "label": "Architect" }, ... ],
    "industries":  [ { "value": "interior", "label": "Interior (incl …)" }, ... ]
  }

Lijsten staan centraal in plugin/profile-options.php (niet in de database).
Zelfde opties als page-edit-profile.php / page-register.php. Bij POST: slug
opslaan als bekend; onbekende waarden pass-through (geen bulk-migratie).

Industry-slugs: interior (niet interior-design) — sluit aan op legacy sector-
waarden. Profession bevat ook professor (alleen in oude shortcode, niet in
register-form).

Audit onbekende meta (rapport alleen, wijzigt niets):
  wp eval 'print_r( md_profile_audit_unknown_meta_values() );'
