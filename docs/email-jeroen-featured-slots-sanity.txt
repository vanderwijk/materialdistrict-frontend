Ik heb de flow zojuist tegen productie gedraaid — dit werkt:

1. GET featured-slots → 0/4, lege slots[]
2. POST boeken (eigen materiaal + maandag ≥7 dagen vooruit) → 201, used=1, status=scheduled
3. Dubbele week → 400/409 "This week is already booked as a featured slot."
4. Ongeldige week_start (geen maandag) → "week_start must be an ISO Monday date."
5. Te kort vooruit (<7 dagen) → "Featured slots must be booked at least 7 days ahead."
6. Plus-account → 403 "Featured placements require a Partner brand tier."
7. DELETE scheduled slot → used terug naar 0
8. /auth/me connected_brands[] → featured_slots_total/used/reset_date kloppen mee

Let op is_featured_now op publiek materiaal-REST:
- wp/v2/material toont is_featured_now alleen als de slot status active is (huidige kalenderweek).
- Geboekte toekomstige week = scheduled → dashboard slot heeft is_featured_now:false; publieke material meta zet het veld dan niet (verwacht).
- Actieve week kun je via POST niet boeken (7-dagen-regel), dus die flip test je het beste met een slot in de huidige week die je in WP zet, of na afloop handmatig de week_start.

  -H 'Content-Type: application/json' \
  -d '{"email":"e2e-dashboard-partner@materialdistrict.com","password":"E2eDashboard2026!"}'

# 2) Membership-velden
curl -sS 'https://materialdistrict.com/wp-json/md/v2/auth/me' \

  | jq '.user.connected_brands[] | {id, slug, tier, featured_slots_total, featured_slots_used, featured_slots_reset_date}'

# 3) Slots lezen
curl -sS 'https://materialdistrict.com/wp-json/md/v2/dashboard/brands/137159/featured-slots' \
  -H 'Authorization: Bearer TOKEN' | jq

# 4) Boeken (week_start = ISO-maandag, ≥ vandaag+7 dagen, vóór reset-datum)
curl -sS -X POST 'https://materialdistrict.com/wp-json/md/v2/dashboard/brands/137159/featured-slots' \
  -H 'Authorization: Bearer TOKEN' -H 'Content-Type: application/json' \
  -d '{"material_id":MATERIAL_ID,"week_start":"2026-06-15"}' | jq

# 5) Annuleren

  -H 'Authorization: Bearer TOKEN' | jq
