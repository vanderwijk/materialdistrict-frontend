## 1. Ruwe response (productie, zelfde backend als test-frontend)

Test-frontend (Vercel) gebruikt `WP_API_URL` → **https://materialdistrict.com/wp-json** (production + preview env op het Vercel-project).

```bash
curl -sS 'https://materialdistrict.com/wp-json/md/v2/material-channels'
```

**Vorm: wrapper, geen kale array**

```json
{
  "channels": [
    { "id": 917, "slug": "acoustic", "label": "Acoustic", "count": 104 },
    { "id": 79, "slug": "biobased", "label": "Biobased", "count": 719 },
    …
  ]
}
```

- **20** items in `channels`
- **`id` komt als JSON-getal** (bv. `79`, niet `"79"`) — in PHP/REST normaal integer
- Velden per item: `id`, `slug`, `label`, `count` (`count` = materialen met dat theme)

Dus: mapper die alleen `Array.isArray(body)` accepteert → lege catalogus → bar alleen “All”. Dat was de bug.
