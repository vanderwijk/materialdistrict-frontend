Hoi Claude,

Helder — je aanpak klopt. Korte bevestiging + thumbnail.

**Filteren**
- Collections: `?theme=<term_id>` op talk, article, brand, event (geen slug in die param).
- Slug→id via `/md/v2/material-channels` is prima; aparte `/wp/v2/theme?slug=` hoef je niet.

**Landingspagina-copy**
- `GET /wp/v2/theme/{id}` → `name` + `description` (core).

**Thumbnail (optie a — live na plugin-deploy)**
- Nieuw REST-veld op het theme-term object: `theme_thumbnail`
- Vorm: `{ id, url, alt }` of `null` als er geen thumbnail in WP admin staat
- Voorbeeld:

```bash
curl -sS 'https://materialdistrict.com/wp-json/wp/v2/theme/79' \
  | jq '{id, name, description, theme_thumbnail}'
```

- `/md/v2/material-channels` blijft `{ id, slug, label, count }` — geen beeld (picker/catalog).

**Plugin**
- Bestand: `taxonomy-theme.php` in materialdistrict-plugin (zelfde patroon als venue-term REST)
- Leest `theme_thumbnail` uit de Tax Meta Class (`tax_meta_{term_id}`), resolved naar attachment-URL

Zodra productie deployed is, kun je landingspagina-kop in één call doen; bars met `?theme=<id>` kunnen parallel.
